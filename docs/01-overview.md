# TaskPing 概要

## 目的

このドキュメントはTaskPingの概要、背景、設計原則を説明する。

## 背景

### 課題

複数のAIコーディングセッション（Claude Code、Codex等）を並列で実行する際、以下の課題がある：

- どのプロジェクトでAIが作業完了したかわからない
- ターミナルを常に監視する必要がある
- 作業内容の把握に時間がかかる

### 解決策

AIセッションが「入力待ち」状態になったタイミングでDiscordに自動通知する。通知には以下を含む：

- プロジェクト名（どこで）
- 作業内容の要約（何をしたか）
- 通知タイプ（成功/エラー/警告/情報）

## 設計原則

### 1. 確定的なトリガー

AIの「判断」ではなく、システムイベント（Stopイベント）をトリガーとする。これにより曖昧さを排除。

### 2. 一方向通知

WebhookベースでDiscordに通知を送信する。双方向通信（Bot）は不要。シンプルさを優先。

### 3. ローカルファースト

要約生成はOllama（ローカルLLM）をデフォルトとする。APIコスト不要、プライバシー確保。

### 4. フェイルセーフ

通知失敗時もClaude Code本体の動作に影響しない。エラーは握りつぶす。

## 機能一覧

| 機能 | 説明 |
|------|------|
| リッチ通知 | Embed形式、色分け（成功/エラー/警告/情報） |
| プロジェクト識別 | Bot名にプロジェクト名を表示 |
| 作業要約 | Ollama/Anthropicによる自動要約 |
| Hooks連携 | Claude Code Stopイベントで自動発火 |
| 自動起動 | Ollama未起動時に自動で起動 |
